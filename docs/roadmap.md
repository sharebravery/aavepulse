# AavePulse 路线图

## 阶段 1：文档与身份重置

目标是从模板生成一个身份独立、结构可追溯的新项目。

- 使用 `gin-admin-cli new` 从 `gin-admin` 生成 AavePulse
- 重命名应用、Go module、构建产物和配置项
- 清理模板演示图片、上游 README、无关配置和产品品牌信息
- 保留并正确处理上游 Apache-2.0 许可证声明
- 补齐项目目标、架构、技术栈和里程碑文档
- 补齐数据库与本地启动约定
- 确定 README 的新叙事

## 阶段 2：后端骨架保留与收敛

目标是保留可用的后台能力，但删掉无关模板内容。

- 保留登录、权限、用户、菜单、日志基础设施
- 建立独立的 `DEFI` 模块并接入模板 Wire 集合
- 建立 Aave 协议、储备市场、快照和同步记录模型
- 建立聚合查询 API

## 阶段 3：The Graph 接入

目标是把 Aave 数据接入跑通。

- 定义可替换的 Graph 查询适配层
- 实现增量拉取和失败重试
- 写入本地数据库
- 形成可展示的指标快照
- 提供显式演示数据模式，外部 Graph 服务不可用时仍能本地验收

## 阶段 4：React 演示前端

目标是让项目可视化、可演示。

- 协议总览页
- 储备市场列表和详情页
- TradingView Lightweight Charts 指标趋势
- 同步状态页

## 阶段 5：博客与发布

目标是把项目包装成可公开传播的作品。

- 整理 GitHub README
- 输出博客正文到仓库外
- 准备截图和演示说明
- 发布到 GitHub

## 后续版本

- 告警规则和告警事件
- 多链 Aave 市场
- 钱包地址风险画像
- Redis 热点缓存和任务状态优化
