# AavePulse 数据库方案

## 结论

开发和演示阶段以本地 `Docker Compose` 跑 PostgreSQL 为必需项。
Redis 作为可选优化项，只有在缓存或任务状态确有需要时再启用。
The Graph 只作为外部数据源，不直接充当数据库。

## 为什么这样选

- 可复现，任何人拉代码后都能本地起环境
- 不依赖线上数据库权限和网络条件
- 适合演示项目，减少部署复杂度
- 先把核心链路跑通，再决定是否引入缓存层
- 后续如果要上线，可以平滑切换到托管数据库

## 组件职责

### PostgreSQL

存放长期数据：

- 协议配置
- The Graph 子图配置
- 同步任务
- 指标快照
- 审计日志

MVP 新增的核心表：

- `defi_protocols`：协议和网络配置
- `defi_reserves`：储备市场当前状态
- `defi_reserve_snapshots`：储备市场历史快照
- `defi_sync_runs`：同步执行记录

告警规则和告警事件属于 P1，不在 MVP 建表范围内。

### Redis

可选的短期缓存层，仅在后续确实需要时启用：

- 任务运行状态
- 热点查询缓存
- 增量同步游标
- 临时聚合结果

## 环境策略

### 本地开发

- PostgreSQL 通过 `docker-compose` 启动
- Redis 默认不启动，除非后续实现明确需要
- 后端本地运行，便于调试
- `.env.example` 只保留变量名，不提交真实值

### 演示部署

- 可以直接沿用同样的容器镜像
- 数据库仍可本地 Docker 或云上托管
- 具体部署方式后续再定，不阻塞当前开发
- Redis 仅在演示性能或缓存收益明确时加入

## 非目标

- 不接入生产数据库账号
- 不在仓库中存放任何真实连接信息
- 不把 The Graph 当作持久化存储

## 数据约束

- 协议由 `protocol + network` 唯一标识
- 储备市场由 `protocol_id + underlying_asset` 唯一标识
- 快照由 `reserve_id + snapshot_at` 唯一标识，重复同步必须幂等
- 金额和利率不能使用二进制浮点直接保存，统一使用数据库定点数
- 同步任务必须记录开始时间、结束时间、状态、读取数量、写入数量和错误摘要
