# English-Only DeFi Interface Design

## Goal

将 AavePulse 前端统一为英文界面，快速消除当前中文、英文和内部字段名混用的问题，同时保留现有 DeFi 专业终端视觉和业务行为。

## Scope

- 不引入 i18n 框架、语言切换控件或多语言资源目录。
- 所有用户可见文案统一使用英文：登录、导航、页面标题、指标、表格、状态、按钮、加载态、空状态、错误提示和同步反馈。
- 保留后端 API、认证流程、演示数据模式、只读约束和 TradingView 图表行为。
- 删除页面组件中的中文用户文案；内部变量名、API 字段名和代码注释不属于界面范围。

## Language And Formatting

- 文案基准为清晰、简短的 DeFi 产品英文，不使用直译式中文英文。
- 日期和时间统一通过 `Intl.DateTimeFormat('en-US', ...)` 输出。
- 金额、数量和百分比统一通过 `Intl.NumberFormat('en-US', ...)` 输出。
- API 状态值通过前端展示映射转换，例如 `succeeded` 显示为 `Succeeded`，`demo` 显示为 `Demo data`。
- 后端未知错误不直接将中文原文展示给用户，统一显示英文兜底提示；已知业务错误映射为稳定英文文案。

## Content Structure

使用轻量的项目内文案模块，不引入运行时翻译依赖。按页面职责拆分为：

- `common`: 通用按钮、加载、刷新、状态和错误兜底。
- `auth`: 登录页、验证码和认证错误。
- `dashboard`: 品牌、网络、只读提示和导航。
- `overview`: 协议概览、指标、市场摘要和数据管道。
- `reserves`: 市场搜索、排序、列标题和空状态。
- `detail`: 资产详情、图表模式、时间范围和风险指标。
- `sync`: 同步记录、执行状态、来源和统计字段。

文案模块只负责静态 UI 文案和状态展示映射，不读取 API、不修改业务状态。

## Error And Empty States

- 保留现有错误边界和请求行为。
- 网络错误、数据加载失败和同步失败显示统一英文提示，并保留可定位的原始错误信息在开发日志中。
- 空列表、未同步和无错误摘要使用明确英文文案，不显示空字符串或内部值。
- 加载按钮、同步按钮和状态徽标在进行中、成功和失败时使用一致的英文状态。

## Acceptance Criteria

1. `web/src` 中不存在面向用户的中文硬编码文案。
2. 登录、概览、储备、详情和同步页面均显示完整英文。
3. 状态值、日期、金额、百分比和数量均使用英文展示格式。
4. API 错误和空状态不会回退为中文或内部字段名。
5. 现有 Tailwind/Radix 视觉、认证、数据加载、图表和响应式布局不回归。
6. 前端单元测试、生产构建和 Go 测试全部通过。

## Verification

- 使用 `rg` 检查 `web/src` 的中文字符和旧的中文文案。
- 添加文案/格式化测试，覆盖状态映射、错误兜底和 `en-US` 日期/数字格式。
- 使用浏览器检查登录页、概览、储备、详情图表和同步记录页面。
