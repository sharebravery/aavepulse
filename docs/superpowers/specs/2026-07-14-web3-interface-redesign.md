# AavePulse Web3 界面重构设计

## 1. 目标

把现有偏传统管理后台的界面，重构为可信、专业、具有链上产品辨识度的 DeFi
数据终端。重构不能停留在深色换肤，必须同时改变组件形态、信息层级、资产表达、
状态反馈、图表容器和响应式导航。

本次只重构前端展示层，不改变现有 API、认证流程、业务数据和后端模块。

## 2. 设计原则

### 2.1 专业，而不是赛博装饰

- 使用深海军蓝背景、电蓝和青绿色数据强调色。
- 发光只用于网络在线、同步中和图表焦点，不给所有边框增加霓虹效果。
- 背景可以有低对比度径向光晕、点阵和链上轨道，但不能干扰数据阅读。
- 不模拟交易下单终端，不增加钱包连接或链上写入暗示。

### 2.2 Web3 气质来自数据结构

- 每个资产都有可识别的 Token Orb，而不是只有文本符号。
- 网络、数据源、只读模式和最近区块状态作为持续可见的协议上下文。
- 利用率、APY、资金规模和同步状态使用不同的视觉编码。
- 数据来源明确标记为 `DEMO DATA` 或 `THE GRAPH`，避免演示数据伪装成实时链上数据。

### 2.3 避免传统后台形态

- 不使用实心矩形侧栏、编号菜单、直角表格和大面积 Ant Design 默认灰色。
- 不把所有内容都放进同一种白底卡片。
- 表格只承担高密度比较；概览和详情优先使用资产卡、指标轨道和数据簇。
- 使用 Radix UI 保留可访问性与交互能力，通过 Tailwind CSS 语义 class 和主题 Token 重塑外观。

## 3. 视觉系统

### 3.1 色彩

| Token | 用途 | 建议值 |
| --- | --- | --- |
| `--void` | 页面最深背景 | `#050816` |
| `--surface` | 一级面板 | `#0B1022` |
| `--surface-raised` | 悬浮面板 | `#11182E` |
| `--line` | 常规边界 | `rgba(151, 170, 255, 0.14)` |
| `--line-active` | 激活边界 | `rgba(102, 126, 255, 0.48)` |
| `--text` | 主文本 | `#F4F7FF` |
| `--muted` | 次级文本 | `#8D99B8` |
| `--blue` | 主交互与供应量 | `#6C7CFF` |
| `--cyan` | 链上数据与图表 | `#37D7FF` |
| `--mint` | 正常状态与正向指标 | `#4EF2C2` |
| `--amber` | 风险利用率与警告 | `#FFBE5C` |
| `--danger` | 错误与高风险 | `#FF668A` |

渐变只用于品牌、Token Orb、图表面积和局部背景光晕。正文和大面积面板不使用
高饱和渐变。

### 3.2 圆角与边界

圆角必须形成层级，不能把所有元素统一成一个值：

- 页面级悬浮侧栏：`24px`
- 一级数据面板和图表容器：`20px`
- 指标卡与资产卡：`16px`
- 输入框、选择器、按钮：`12px`
- 状态胶囊和标签：`999px`
- Token Orb：圆形

面板使用一像素半透明边界和内高光，不使用传统后台的硬描边、投影偏移和纸张阴影。
悬浮状态通过边界变亮、背景抬升和轻微位移表达，不使用大范围阴影。

### 3.3 字体与数字

- 正文与中文使用 `Fira Sans`，保证现有依赖不扩张。
- 数字、地址、网络和状态使用 `Fira Code`。
- 标题不再使用超大的中文海报字号；桌面主标题控制在 `36-44px`。
- 财务数字使用等宽字体和紧凑字距，单位弱化显示。

### 3.4 动效

- 页面进入采用 360-480ms 的淡入和上移，面板按阅读顺序轻微错峰。
- 网络在线点与同步中状态使用低频脉冲。
- 背景光晕缓慢漂移，周期不少于 12 秒。
- Token Orb 悬浮只允许 2-3px 位移和边界增强。
- 遵守 `prefers-reduced-motion`，关闭非必要动画。

## 4. 页面结构

### 4.1 应用外壳

桌面端采用悬浮侧栏加工作区，而不是贴边的传统后台导航。

侧栏包含：

- AavePulse 品牌标识和 `DeFi Intelligence` 副标题。
- Ethereum 网络胶囊，显示在线状态和 `Mainnet`。
- 三个主导航项，使用图标、标题和短说明，不再显示 `01/02/03`。
- 底部固定显示 `Read-only`、数据源和退出入口。

工作区顶部增加窄状态栏，显示网络、数据源、最近同步时间和只读模式。页面标题降级为
内容标题，避免每页都像传统后台模块首页。

移动端取消顶部压缩侧栏，改成底部浮动导航。网络和数据源进入页面顶部状态区。

### 4.2 登录页

登录页采用单一沉浸式深色画布，登录面板悬浮在右侧，不再左右切成两块企业宣传页。

- 背景使用低对比点阵、数据轨道和两个缓慢移动的协议光晕。
- 左侧展示品牌主张、只读安全说明和三项能力：The Graph、90 天历史、零链上写入。
- 登录面板使用 24px 圆角玻璃容器和内高光。
- 输入框使用深色填充、12px 圆角和清晰焦点环。
- 验证码作为输入框右侧独立胶囊，可点击刷新。
- 演示账户提示弱化为面板底部说明，不抢占标题层级。

### 4.3 协议概览

概览首先回答协议规模、资金效率和数据是否可信。

- 顶部 Hero 显示 `Aave V3 / Ethereum`、总供应量和最近同步状态。
- 四项核心指标使用非对称 Bento 布局；总供应量占更大面积，其余指标紧凑排列。
- 指标卡包含短标签、数值、语义图标和微型装饰轨道，不再使用编号水印。
- 核心储备从普通表格改为六张资产行卡：Token Orb、符号、名称、供应量、利用率轨道。
- 同步信号改为管道状态卡，包含数据源、最近运行、读写数量和成功状态，不再使用旋转圆环。

### 4.4 储备市场

- 顶部使用搜索、排序和市场数量组成一条控制带。
- 桌面端保留高密度表格，但去掉 Ant Design 默认表头和分割线观感。
- 每行包含 Token Orb、资产信息、供应/借款量、可用流动性、利用率轨道和 APY 胶囊。
- 行悬浮时显示进入箭头并强化边界，整行可点击。
- 移动端不横向压缩表格，改为资产卡列表，展示最关键的供应量、利用率和 APY。

Token Orb 由资产符号稳定映射到渐变色和首字母，无需引入外部 Token Logo 服务，
从而保证离线演示和构建稳定。

### 4.5 储备详情

- 资产头部包含大号 Token Orb、名称、符号、截断后的合约地址和网络标签。
- 四项指标改为独立数据卡，利用率使用风险颜色轨道。
- 图表面板使用 20px 圆角、深色绘图区和顶部控制带。
- 流动性/利率切换与 `7d/30d/90d` 使用分段胶囊，当前项带柔和高光。
- TradingView 供应曲线使用电蓝，借款曲线使用青绿或琥珀；背景、网格和文字同步深色主题。
- 保留 `Charting by TradingView` 署名。

### 4.6 同步记录

- 顶部展示最近同步状态、来源、耗时和写入量摘要。
- 历史记录使用时间线式数据表，状态以带图标的胶囊呈现。
- `执行同步` 是唯一主按钮；执行中显示状态脉冲并禁用重复提交。
- 错误摘要保留完整可访问文本，界面中可截断但提供悬浮提示。

## 5. 组件边界

重构新增以下纯展示组件，避免把全部视觉逻辑继续堆在页面文件中：

- `TokenOrb`：根据资产符号生成稳定的颜色与缩写。
- `ProtocolStatusBar`：网络、数据源、同步时间和只读状态。
- `MetricCard`：统一指标的标签、数值、语义色和辅助信息。
- `UtilizationBar`：统一利用率轨道与风险色。
- `StatusPill`：统一网络、同步和数据源状态。
- `PageIntro`：统一页面标题、说明和操作区。

组件只接收展示数据，不自行请求 API。页面继续负责加载、错误和交互状态，现有
`lib/client.ts` 与后端契约保持不变。

## 6. 状态与错误处理

- 加载状态使用与最终布局一致的深色骨架屏，避免白色闪烁。
- API 错误使用面板内错误状态和重试按钮，不跳回空白页。
- 无数据状态说明数据源、筛选条件和下一步操作。
- 同步期间按钮和状态栏同步显示执行状态。
- Demo 数据在所有核心页面持续可见，不只在概览页显示一次。

## 7. 响应式与可访问性

- 桌面宽度 `>= 1100px` 使用悬浮侧栏和 Bento 布局。
- 平板宽度 `768-1099px` 缩小侧栏，指标变为两列。
- 移动宽度 `< 768px` 使用底部导航、单列卡片和移动资产列表。
- 所有正文与背景达到 WCAG AA 对比度。
- 焦点状态不能只依赖颜色，交互元素保留清晰轮廓。
- Token Orb 和装饰背景不替代文本信息。

## 8. 验收标准

1. 登录、概览、储备列表、储备详情和同步记录统一为 DeFi 专业终端视觉。
2. 视觉变化包含布局、组件结构、圆角、边界、资产标识、状态编码和动效，不是单纯换色。
3. 页面不再出现编号侧栏、纸张背景、硬边框卡片和默认 Ant Design 表格观感。
4. 桌面端 1440px 和移动端 390px 均无内容溢出或关键操作遮挡。
5. TradingView 图表完整适配深色主题，切换模式和时间范围正常。
6. 现有认证、筛选、排序、详情跳转和同步功能保持不变。
7. 前端测试和构建通过，浏览器控制台无新增错误。

## 9. 非目标

- 不增加钱包连接、签名、交易或合约写入。
- 不接入远程 Token Logo 服务。
- 不引入 3D、WebGL 或重型动画依赖。
- 不改变后端 API、数据库结构或权限模型。
