# AavePulse 项目目标

## 项目定位

这是一个由 `gin-admin` 模板演化而来的 Aave DeFi 数据监控项目。
项目重点不是做完整协议，而是做一个可展示、可讲解、可发 GitHub 的只读后台系统。

项目要回答的问题很具体：

- Aave 的关键链上数据如何被稳定索引和聚合
- 后台如何对协议、指标、告警、同步任务进行管理
- 前端如何用一个简单的演示页把结果展示出来

## 核心目标

1. 使用 `gin-admin` 作为工程底座，保留权限、日志、Swagger 和 Wire 能力。
2. 引入 The Graph 作为 Aave 历史链上数据来源。
3. 做出一个最小可用的 Web3 数据管理后台。
4. 用同步任务、持久化和聚合查询体现完整后端工程能力，而不是只做代理 GraphQL 请求。
5. 输出一套适合博客传播的设计文档和技术叙事。

## 面向读者

- 想了解 Aave 和链上数据监控的 DeFi 学习者
- 想看 Go 后台如何接入 The Graph 的开发者
- 想参考一个“快速出成果”的 Web3 演示项目的人

## 成功标准

- 仓库克隆后，读者一眼能看出项目是做什么的
- 后端有明确的数据接入、同步、聚合、权限和审计边界
- 前端可以展示协议概览、指标图表和同步状态
- 项目可以通过本地 Docker PostgreSQL 和明确的启动命令复现
- 项目品牌、包路径、配置和运行信息统一为 AavePulse
- 保留上游许可证要求，但不在产品界面和项目介绍中沿用模板品牌
